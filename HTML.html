HTML Foundations

Semantic HTML makes every layer of your site stronger.

Use semantic tags to communicate meaning, not just layout. The result: better accessibility, clearer structure, and stronger SEO.

MDN HTML Reference

Core Document Skeleton

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Page Title</title>
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
  </body>
</html>

Structure

Semantic layout map

Element Use it for Tip
<header> Branding, navigation, intro content Keep it consistent across pages
<main> Primary page content Use only one per page
<section> Grouped content with a heading Great for feature blocks
<article> Self-contained content Ideal for blog posts
<aside> Secondary information Use for tips or sidebars
<footer> Summary, links, legal info Keep it lean and useful

Forms

Forms that feel pro

Combine semantic inputs, labels, and validation attributes for clean UX and reliable data.

  • Always connect <label> with for.
  • Use type and autocomplete hints.
  • Add required and minlength to help users.
<form>
  <label for="email">Email</label>
  <input id="email" type="email" required />
  <label for="role">Role</label>
  <select id="role">
    <option>Designer</option>
    <option>Developer</option>
  </select>
  <button type="submit">Send</button>
</form>

Accessibility

Accessible by default

Quick checklist

  • Meaningful heading order (h1 → h2 → h3).
  • Alt text that explains the purpose of images.
  • Buttons for actions, links for navigation.
  • Visible focus states for keyboard users.

Landmarks matter

Assistive tech uses landmarks like <nav>, <main>, and <footer> to help people jump around a page quickly.

Combine landmarks with descriptive link text to make navigation easy.
