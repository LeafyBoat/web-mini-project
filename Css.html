CSS Strategy

Design with tokens, layout with intent.

CSS is where your interface gains clarity and personality. Think in systems: consistent spacing, scale, and reusable patterns.

MDN CSS Reference
Design Tokens

Define reusable values

:root {
  --space: 1.5rem;
  --radius: 16px;
  --accent: #f05d3b;
}

Cascade

Specificity in one glance

/* Lowest → Highest */
button { }
.button { }
header .button { }
#hero .button { }

Keep specificity shallow to make your styles easy to override and maintain.

Layout

Flex + Grid essentials

  • Use Flexbox for one-dimensional layouts.
  • Use Grid for two-dimensional structure.
  • Combine both for responsive dashboards.
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 1.5rem;
}

.stack {
  display: flex;
  gap: 1rem;
  align-items: center;
}

Responsive

Design for the smallest screens first

Use fluid units and media queries to adapt layouts without rewriting everything.

  • Prefer rem and % for scalable sizing.
  • Set max widths to keep content readable.
  • Use min() and clamp() for responsive text.
h1 {
  font-size: clamp(2rem, 2vw + 1.5rem, 3.5rem);
}

@media (min-width: 900px) {
  .layout { grid-template-columns: 2fr 1fr; }
}

Polish

Depth, motion, and micro contrast

Motion guidelines

  • Use transitions for feedback, not decoration.
  • Animate only transform + opacity when possible.
  • Keep durations under 300ms for UI feedback.
.card {
  transition: transform 200ms ease, box-shadow 200ms ease;
}

.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 20px 30px rgba(0,0,0,0.12);
}

Next Step

CSS pro checklist

  • Design tokens defined for color, spacing, and typography.
  • Layouts use Flex/Grid with consistent gaps.
  • Components have hover and focus states.
  • Responsive behavior tested on small screens.
  • Buttons and links have clear focus styling.
